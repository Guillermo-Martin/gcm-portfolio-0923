Password generator

I redesigned the Password Generator from my bootcamp to provide a better user experience (WORK ON THIS WORDING)

Why I did this project/Project Overview

Summary

After working for 2 years as a junior web developer, I wanted to challenge myself and see if I can improve upon one of my bootcamp projects.

The goal

My goal in this project was to see if I can change the design and the functionality in order to provide a better user experience.

First steps

Identifying opportunites for improvement/pain points

One of the first things I did, was go through the password generator and identify some pain points. Here are some of the pain points I identified:

  • - With the exception of the prompt asking for password length, if a user doesn't select any password options, a blank password is provided to the user
  • - The questions are asked sequentially. If a user changes their mind, they're unable to go back to select the option they passed on. The only way they could go back is to generate a new password.
  • - The generator uses prompts to ask what the user wants in their password.
  • - The generator lacks a copy feature

Challenges

Challenge #1

CHALLENGE: determining which options were selected now that checkboxes were used and used an object to keep track of which options were selected

SOLUTION: accomplished by looping through an object

Challenge #2

CHALLENGE: after switching to checkboxes, and creating a password, the final password length was actually multiplied by however many options were selected

SOLUTION: solved by adjusting the if statement; the if statement was very strict in that if the user’s password length didnt’ match the lenght of the password as it was being generated, then my code would loop creating a very long password. It’s possible that the as the password was being generated during one of the character checks, the number was not equal, so the function kept adding characters until the count variable in the for loop was greater than the password length.

Challenge #3

CHALLENGE: PREVENT A USER FROM SELECTING MORE OPTIONS AND CREATING ANOTHER PASSWORD THAT WOULD ADD MORE CHARACTERS TO AN ALREADY CREATED PASSWORD

SOLUTION: to solve this, I disabled the options and removed the “create password” button after a user created a password. This also improved the user experience and also allows a user to create another password if they so choose to.
