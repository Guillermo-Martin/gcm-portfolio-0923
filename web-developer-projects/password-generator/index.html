Password generator redesign

A redesigned password generator based off an old bootcamp assignment.

Project Overview

Summary

After working for two years as a junior web developer, I wanted to challenge myself and see how much I've grown as a developer. I decided to revisit one of my earliest projects, an assignment from my bootcamp - a password generator. In this application, a user can create a password between 8 and 128 characters in length, based on prompts given to a user.

The goal

My goal for this project was to go through the original project, identify opportunities for improvement, and redesign the application and its functionality to provide a better user experience.

First steps

Identifying opportunites for improvement

One of the first things I did, was go through the password generator and identify some opportunities for improvement. To see what the original application looked like, click here. In going through the application, I found a few things that could be improved:

Blank password

With the exception of the prompt asking for password length, if a user doesn't select any password options (they select "Cancel" in all the prompts), a blank password is provided to the user.

An inability to go back if a user changes their mind

The questions asking what a user wants in their password are asked sequentially. If a user changes their mind, they're unable to go back to the option they passed on. The only way they could go back is to finish creating a password and then create another one, or refresh the page and start over.

The overall design

The design of the application was already made, as the assignment mainly focused on creating the functionality. The application also uses JavaScript prompts to ask what the user wants in their password.

Lack of a copy feature

The generator lacks a copy feature so a user would have to manually copy their password.

Tackling these problems

The overall design

In coming up with the new design, one of the things I wanted to address was the default Javascript prompts. Thinking about the user and how the prompts didn't allow a user to go backwards, I opted to use checkboxes so a user could select and unselect their options. The checkboxes informed how I was going to design the rest of the application. With the checkboxes in mind, I created a few wireframes and added stars to the particular sections that I liked.

SEQUENTIAL QUESTIONS/UNABLE TO GO BACKWARDS

With the checkboxes now part of the UI, a user can freely select their password options without the constraints provided by the Javascript prompts.

BLANK PASSWORD

WORK ON WORDING - I added warning messages for the user and prevented them from making a password until they selected at least one option.

To address the blank password problem, I added some "if" statements that check to see if at least one options was selected. If none of the options are selected a warning message appears to the user and a password isn't made.

COPY FEATURE

In order to add the copy feature, I used the Clipboard API. I added an event listener on the password box, that when clicked, would copy the password box contents to the clipboard. When testing out the application, I found out the event listener was active and a user could copy the contents of the box, even if it was empty. In order to mitigate this issue, I added an "if" statement to check whether or not the password box was empty. If it was, then the copy function won't work, preventing a user from copying an empty string. I also added some feedback to let a user know that their password was copied.

Challenges

Challenge #1 - Determining which of the checkboxes were selected

Now that the generator uses checkboxes, my solution involved the following steps:

  1. Check to see which boxes were checked using the ".checked" property
  2. Create an object to hold the password options
  3. Loop through all of the checkboxes, add the information, key and value, to the object
  4. Next, after the passwordOptions object was created, loop through the object using a "for...in" loop to see which options are "false"
  5. If an option is "false", add one to a counter (for keeping track) (created a variable)
  6. If the counter equals 4, no options were checked so warn the user

The code runs whenever the "Create password" button is clicked.

Challenge #2 - Incorrect password length

After switching to checkboxes, and creating a password, the final password length was actually multiplied by however many options were selected. In the GIF, the password length is noted in the console at the end of the password. In this example, if the password length is 128 characters in length and 4 options are selected, the final password is 512 characters in length; if the password length is 128 and 2 options are selected, the final password is 256 characters in length.

The problem was in the "if" statement. The if statement was very strict in that if the user’s password length didn't match the length of the password as it was being generated, then my code would loop creating a very long password. It’s possible that the as the password was being generated during one of the character checks, the number was not equal, so the function kept adding characters until the count variable in the for loop was greater than the password length.

The conditional was revised so that if the password was greater than the password length, the generator stops adding characters.
