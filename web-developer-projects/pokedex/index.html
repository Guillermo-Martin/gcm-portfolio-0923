Pokédex

Using PokéAPI to obtain data, this application allows users to get basic information on Pokémon.

Project Overview

Summary

Utilizing the PokéAPI and the “.fetch()” JavaScript method, this application dynamically creates and renders information on Pokemon.

The goal

The goal for this project was to display Pokémon information to a user in a dynamic and engaging way.

Steps to creating the Pokédex app

Determine what type of data I obtain from the PokéAPI and decide the type of data I’d like to display.

I used the “.fetch()” JavaScript method and “console.log()” to determine the data I would get and how I would access certain data.

Get the basic functionality working.

A user enters a Pokémon name in the input and submits the form. Once data is retrieved, the Pokémon's sprite and basic information is displayed on the screen.

Design the app and determine how the information should be displayed.

I created wireframes and created high-fidelity mockups in Figma. Feel free to take a closer look at the designs! →

Start building!

I built this project using HTML, CSS, and JavaScript. I used Greensock for the animated elements.

Challenges

Challenge #1 - Getting the animation to play correctly

In order to get the color animation to play correctly, I found that breaking the animation down into steps was the easiest way to accomplish this:

  1. Create rectangles using "div" elements, position them on the page, and set their background color to more easily see the animation.
  2. Set the transform-origin for the colored rectangles in CSS. By setting this in CSS, it made the entrance effect easier to accomplish using Greensock's ".to()" method.
  3. Set "scaleX" to 0 for the rectangles using Greensock. By doing this, the width of the rectangles is 0, allowing for expansion of the rectangles later on.
  4. Increasing the rectangles to full width using Greensock's ".to()" method. The rectangles go to full width by setting "scaleX" to 1.
  5. After getting the animation to work correctly, I then rotated and positioned the rectangles on the corners of the page according to the high-fidelity mockups I created earlier.

Challenge #2 - Getting the audio to play

I wanted to add an option to listen to a Pokemon’s cry. I learned about the Audio constructor in Javascript. To get this to work, I decided to add a click listener on the sprites. When a user clicks on one of the sprites, a new Audio element is created and the sound file is passed as an argument (which was obtained from the Pokemon data obtained from the API call). The “.play()” method is then used to play the cry.

An issue I ran into while trying to get this functionality to work, was that even though the sound played correctly, I got the following error: “Uncaught (in promise) DOMException: The ".play()" request was interrupted by a new load request. https://goo.gl/LdLk22”. This error would only show after searching for subsequent Pokemon and playing their cry.

I learned that the “.play()” method is asynchronous and returns a promise. The code I originally wrote wasn’t handling the promise, so I was getting that error. The link provided in the error showed a solution, which I implemented in my own code.

Challenge #3 - An edge case: Pokémon with punctuation in the name

I discovered this issue purely by accident. As I was searching for Pokemon, I decided to mix up my search and search for “Mr. Mime.” When I did, no results came up, which was unusual, given that was the name of that Pokemon.

I did some research and found that the name used when making an API call used a hyphen instead of a period. So, in order get that particular result, “Mr. Mime” would have to be “mr-mime.” While trying to resolve this edge case, I found that, in general, Pokemon with some sort of punctuation in their name didn’t produce any results when searched for.

In order to resolve this, I searched for Pokemon names with some sort of punctuation in their name. I created a switch statement in order to evaluate different variations of a Pokemon name users may search for. If any of those variations came up, the input used to make an API request would change to an input that would return a result. In this case, any variations of “Mr. Mime” would be changed to “mr-mime.”

Learnings and future

Learnings

Working on this project I learned about changing the transform-origin of an element. I also learned about the importance of breaking down more complicated tasks into smaller digestible pieces. Finally, I learned about working with the “.fetch()” JavaScript method and working with APIs.

Future

If I were to continue working on this project, I'd do the following:

  • Some Pokémon have different forms, depending on the region. Future iterations could also display these different forms.
  • Conduct some user research to see what information is most useful and find ways to display additional information.
